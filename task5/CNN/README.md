# CNN 学习笔记

## im2col

实际上`im2col`的过程很简单，将每个窗口展平，然后堆叠成一个形状为(H_out * W_out, K * K)的矩阵，但这个方法非常的巧妙，可以很好的简化前向传播和反向传播，本质是将卷积层特殊的线性运算转换为一般的线性运算。

## 反向传播

首先要理解前向传播时，我们的目的仅是为了得到一个值（损失），而反向传播并非简单的传回损失对输入的梯度，中间会有许多的分叉，分别是损失对每个参数的梯度。

### 卷积层

卷积层的反向传播在整个网络中是最复杂的那一批，纯数学推导的话简直是地狱。但如果仔细想想，就会发现整个卷积层的操作无非是一次有点特殊的线性运算，而一般的线性运算的梯度是非常好求的，从这个角度上看，要求`dW`就是要找出每个`w`对应了哪些`x`。通过具体例子，不难发现，每个`w`对应的刚好是一个个小窗口，类似于前向卷积。这样一来，卷积层的反向传播就非常好求了，结合`im2col`的思想，求梯度就变成了凑形状。

+ X_col: (N, C_in, H_out * W_out, K * K)

+ dy: (N, C_out, H_out, W_out)

+ W: (C_out, C_in, K, K)

这是已知数据的形状，我们的目标是`dW`，其形状应该和`W`一致，所以我们将`X_col`转化为(N, H_out * W_out, C_in * K * K)，将`dy`转化为(N, C_out, H_out * W_out)，二者做矩阵乘法得到(N, C_out, C_in * K * K)，这已经和我们的目标很接近了，只要在进行一次变形，最后对 N 求和即可。

同理，`dX`也可以这样去凑形状，这样的方法之所以成立，本质是因为卷积层只进行了线性运算，再结合`im2col`，就可以断定最后的梯度公式一定可以化成一次简单的矩阵乘法，所以说`im2col`很重要。

### 池化层

池化层是一个非线性层，包含了`max`函数，所以不难想到将它和 ReLU 层对比，同样地，池化层的反向传播也是在ReLU的基础上凑出正确的形状。

### BatchNorm

BN的反向传播可谓是非常复杂，但是由于BN的完全是初等函数，所以老老实实数学推导就行，虽然麻烦，但并不算难。
